04 / 08
● WORKFLOW · 4 STEPS

From idea to delivery, one workflow

1

Pick a starting point

Describe your goal in one sentence, or start straight from a template or plugin.

2

Lock the visual direction

Once a direction is chosen, palette, type, and spacing flow into generation automatically.

3

Generate artifacts

The agent reads the full context and produces real, runnable files — preview and edit instantly in the sandbox.

4

Ship, or turn it into video — memory compounds

Hand off to engineering, or convert to a marketing video with HyperFrames; every choice becomes context for the next generation.

● LOCAL DATA

Your data stays on your machine

// .od/ — all of Open Design's runtime data lives in this local directory
.od/
├── app.sqlite          // local SQLite: sessions & design memory
├── projects/           // per-project agent working directories
├── artifacts/          // generated prototypes / slides / videos
└── media-config.json   // BYOK credentials, API keys never leave the machine

No sign-up, no cloud sync — git it, diff it, roll it back. Design artifacts are finally version-controlled.
