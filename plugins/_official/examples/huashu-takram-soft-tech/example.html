01 · Philosophy
Design Philosophy

Put frontier AI design power
back on every creator’s desk.

Claude Design proved an LLM can truly design — yet it is closed, paid, and cloud-only. Open Design sets that capability free.

Desktop Native
Design happens on the desktop

Local files, Figma exports, and code repos are directly readable — and agents hold the full power of terminal execution.

Agent Integration
Don’t build agents — plug them in

Claude Code, Codex, and Cursor on your machine are already strong; Open Design wires them into a complete design workflow.

Design Memory
Learns you over time

Every choice settles into design systems, preferences, and memory — so the next generation lands closer to what you want.

04 · Architecture
Platform Architecture

Three layers: Agents — Workflow — Artifacts

Agent layer

Auto-detects the CLIs already on your PATH — Claude Code, Codex, Cursor, and 21 coding agents in all, zero configuration.

Workflow layer

217+ plugins and 129 design systems supply starting points and context; agents produce real, runnable files inside a sandbox.

Delivery layer

Prototypes, slides, posters, and dashboards ship straight to engineering — or become HTML video via HyperFrames.

05 · Workflow
Workflow · Idea to Delivery

From idea to delivery

Step 01 · Start
Pick a starting point

Describe the goal in one sentence, or start straight from a template or plugin.

Step 02 · Direction
Set the visual direction

Once a direction is chosen, palette, type, and spacing carry into every later generation.

Step 03 · Generate
Generate the artifact

The agent reads the full context and produces real, runnable files — instant preview and edits in a sandbox.

Step 04 · Ship
Ship & remember

Export to engineering, or turn it into a marketing video with HyperFrames; every choice becomes memory for the next run.
